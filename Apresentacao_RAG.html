Este documento apresenta o relatório detalhado do que foi feito e os locais exatos onde as implementações foram realizadas para concluir com êxito a 3ª Etapa: Consulta com RAG (Retrieval-Augmented Generation) do projeto Administrativo Financeiro.

O que foi desenvolvido?

Foi criada a estrutura completa de busca utilizando Inteligência Artificial acoplada ao Banco de Dados SQLite local por meio da técnica de RAG. Foram disponibilizados dois métodos distintos de busca para o usuário:

🔍 RAG Simples (Keyword Match)

Recupera os registros do banco de dados convertidos em formato textual com base na correspondência direta de palavras-chave extraídas da pergunta (removendo stop words do português). Ordena os resultados pelo maior número de correspondências antes de submeter ao modelo de linguagem Gemini 2.5 Flash.

🔮 RAG Embeddings (Busca Semântica)

Gera e armazena vetores numéricos de 3072 dimensões (embeddings) para cada registro e para a pergunta usando a API do Gemini (modelo gemini-embedding-2). Efetua a similaridade de cosseno em memória para encontrar as correlações semânticas mais próximas, oferecendo respostas precisas mesmo que os termos de busca não sejam idênticos aos cadastrados.

⚡ Sistema de Cache em Banco de Dados

Para garantir máximo desempenho e economia de tokens da API do Gemini, foi desenvolvida uma tabela de cache no SQLite (DOCUMENT_EMBEDDINGS). Os embeddings vetoriais de cada registro do sistema são gerados apenas uma vez e reaproveitados nas consultas subsequentes. O cache é sincronizado e atualizado dinamicamente caso o registro sofra alterações ou novas notas sejam integradas.

Estrutura do Prompt de Instrução da LLM

A IA foi configurada para atuar estritamente como um assistente financeiro baseado em dados. O prompt força o modelo a responder apenas com o contexto fornecido pelo RAG:

Você é um assistente financeiro inteligente e analítico do sistema Administrativo Financeiro. Seu objetivo é responder à pergunta do usuário baseando-se estritamente nos dados do banco de dados contidos no CONTEXTO abaixo. Regras: 1. Baseie sua resposta APENAS nos dados informados no contexto. Não invente ou presuma informações. 2. Caso o contexto não contenha informações suficientes, diga que não encontrou essa informação. 3. Formate valores monetários em R$ e datas em DD/MM/AAAA.
Demonstração de Funcionamento (Resultados dos Testes)

Durante os testes integrados locais, o sistema respondeu corretamente às consultas baseadas em dados inseridos no banco:

  • Pergunta: "Quais despesas eu tive com manutenção ou operação?"
    Fontes Recuperadas (SQLite - Busca Semântica por Cosseno): Movimentação ID 9 (Agropecuaria São João, Categoria: MANUTENÇÃO E OPERAÇÃO, Valor: R$ 320.00)
    Resposta da IA: "Identifiquei a seguinte despesa nesta categoria: Fornecedor Agropecuaria São João no valor de R$ 320,00."
